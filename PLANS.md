# План розвитку NeonRust Pro (RustOpti)

Цей документ містить усі поточні завдання щодо компіляції, захисту та релізу програми.

## 1. Створення інсталятора (Build Process)
Для отримання готового `.exe` або `.msi` файлу:
- Фронтенд: `npm run build` (Вже перевірено ✓)
- Бекенд: `cargo check` (Вже перевірено ✓)
- **Збірка інсталятора**: 
  ```cmd
  set "CARGO_BUILD_JOBS=1" && npm run tauri build
  ```

## 2. Покращення безпеки (Security Upgrade)
Ми реалізуємо 4 рівні захисту для комерційного релізу:

### А. Мережевий захист
- **Заміна CURL**: Використання бібліотеки `reqwest` замість зовнішніх команд.
- **HTTPS**: Переведення всіх запитів на захищений протокол.
- **Валідація**: Підпис відповідей сервера для запобігання підміні даних.

### Б. Захист бінарного файлу (Hardening)
- **Strip Symbols**: Видалення імен функцій та відлагоджувальної інформації.
- **LTO (Link-Time Optimization)**: Глибока оптимізація коду для швидкості та ускладнення декомпіляції.

### В. Anti-Debugger (Захист від зламу)
- Додавання коду, який перевіряє, чи не намагаються "зареверсити" програму в реальному часі.

### Г. Hardware ID (HWID)
- Покращення генерації унікального коду комп'ютера для прив'язки ліцензії.
